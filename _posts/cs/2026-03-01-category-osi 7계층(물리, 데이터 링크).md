>## 개념

- OSI 7계층은 네트워크 통신을 기능 단위로 계층화한 참조 모델
- 각 계층은 독립적인 역할을 부여 받아 동작하며, 문제 발생 시 현상에 따라 원인 파악이 용이함
- 국제표준화기구인 ISO가 제안한 모델이며, 실제 프로토콜 구현의 기준 아키텍처로 사용됨

![](https://cdn.promleeblog.com/posts/250514-142046.webp)

---

>## 물리 계층

- 2진수의 데이터를 전기/광/무선 신호로 바꾸는 계층
- 비트(bit, 0 또는 1)의 통신단위로 통신
- 신호의 전압, 타이밍, 전송 속도 등을 정의
- 이 계층에 속한 대표적인 장비는 케이블, 허브, 리피터 등이 있다.

---

>## 데이터 링크 계층

- 같은 네트워크 구간 내에서 **프레임 전달**을 담당하는 계층
    - **프레임**: 송 수신 MAC 주소 + 데이터 + 오류 확인 값의 묶음
- 스위치를 이용하여 같은 네트워크 내부에서 MAC주소를 기준으로 프레임을 전달

---

>## 결론

- 물리 계층은 신호를 보낸다.
- 데이터 링크 계층은 그 신호에 누구에게 보낼지를 붙인다.

| 구분 | 물리 계층 (1계층) | 데이터 링크 계층 (2계층) |
| --- | --- | --- |
| 역할 | 비트를 신호로 변환하여 전달 | 같은 네트워크 내부에서 프레임 전달 |
| 다루는 단위 | Bit (0과 1) | Frame |
| 주소 개념 | 없음 | MAC 주소 |
| 판단 기준 | 신호 유무 | 목적지 MAC 주소 |
| 주요 기능 | 전압, 타이밍, 속도, 케이블 규격 정의 | 프레이밍, MAC 기반 전달, 오류 검출 |
| 오류 처리 | 없음 | 오류 검출 (복구는 하지 않음) |
| 대표 장비 | 케이블, 허브, 리피터 | 스위치 |
| 통신 범위 | 단순 신호 전송 | 같은 LAN / 같은 VLAN 내부 |
